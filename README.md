# Electricity Theft Detection (theft_detection.ipynb)

This repository contains a single Jupyter notebook (`theft_detection.ipynb`) that implements a complete pipeline for detecting electricity theft from hourly consumption data. The notebook generates synthetic consumption data with injected theft patterns, engineers features, trains multiple detection models (attention-based autoencoder, LSTM forecaster, XGBoost / Random Forest / IsolationForest), and combines them with a rule-based engine in an ensemble to produce consumer risk scores.

## What is included

- `theft_detection.ipynb` — Main notebook implementing the end-to-end pipeline:
  - Synthetic data generation with multiple theft patterns
  - Exploratory data analysis and visualizations (saved to `figures/`)
  - Feature engineering (sequence-level stats and temporal features)
  - Preprocessing, scaling, and train/val/test splits
  - Attention-based autoencoder (anomaly detection)
  - LSTM forecaster (sequence prediction)
  - Rule-based detection engine (expert rules)
  - Ensemble fusion and threshold analysis
  - Traditional ML baselines (XGBoost, Random Forest, Isolation Forest)
  - Outputs and reports saved to `outputs/` and `report/`

## Produced files (when the notebook is run)

- `synthetic_consumption.csv` — Synthetic dataset generated by the notebook
- `figures/` — Visualizations (EDA, training curves, comparisons)
- `models/` — Trained model artifacts (.h5, .joblib)
- `scalers/` — Saved scalers for preprocessing
- `outputs/consumer_risk_scores.csv` — Final consumer risk scores
- `outputs/evaluation_results.json` — Evaluation summary and metrics
- `outputs/pipeline_artifacts.json` — Pipeline summary and artifact listing
- `report/` — Executive summary (notebook may generate `report/executive_summary.md`)

## Run instructions

Option A — Run in Google Colab (recommended for GPUs):
1. Upload `theft_detection.ipynb` to Colab.
2. In Colab runtime: Runtime > Change runtime type > Hardware accelerator: GPU (optional).
3. Run cells from top to bottom. The notebook contains install commands for missing packages.

Option B — Run locally (Windows / PowerShell):
1. Create and activate a virtual environment (PowerShell):

```powershell
python -m venv .venv; .\.venv\Scripts\Activate.ps1
```

2. Install requirements:

```powershell
pip install -r requirements.txt
```

3. Start Jupyter and open the notebook:

```powershell
jupyter notebook
```

4. Run cells sequentially. Some long training steps may take significant time on CPU.

## Notes and recommendations

- The notebook generates synthetic data (`synthetic_consumption.csv`) — no real customer data is included.
- For production use, replace synthetic data with your real meter readings and review the feature engineering accordingly.
- Use GPU for faster training of TensorFlow models (autoencoder and LSTM). On local machines, install TensorFlow with CUDA support matching your GPU/driver.
- Before pushing to a shared repo, consider removing large model artifacts or adding Git LFS for model files.
